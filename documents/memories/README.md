# Admin Panel Memories & ADR (Architectural Decision Records)

This directory serves as the "Long-term Memory" for the ZaaZ Admin Panel project. It is used to store the reasoning behind major architectural decisions, developmental milestones, and "contextual history" for complex logic.

---

## 📂 Directory Structure

1.  **`00x-filename.md`**: Numbered Architectural Decision Records (ADR).
2.  **`001-ui-framework-selection.md`**: Standard UI framework selection (Vite + React + Tailwind + shadcn/ui).
3.  **`002-state-management.md`**: Standard state management selection (Redux Toolkit + RTK Query).

---

## 🧠 Why use this?

- **Context:** Code shows *how*, Memories show *why*.
- **Onboarding:** New developers can quickly understand why certain constraints exist.
- **Legacy Avoidance:** Prevents accidental "re-solving" of problems that were already addressed.
- **Consistency:** Maintains the architectural vision over time.
